### D. `<Blocker>` direct-attack gating

`<Blocker>` is a keyword declared in card data, but exercising "rest this unit
to change the attack target to it" needs the declareBlock move to be
combined with an attack fixture where a direct attack is redirected onto the
blocker. No card-level test does that today (the engine's battle-phase
`describe("Battle Phase — TODO: Complex scenarios")` block explicitly flags
it as a TODO — see `it.todo("Blocker keyword …")`).

Representative cards:

- `unit/079-rick-dias.ts`
- `unit/006-forbidden-gundam.ts` (Blocker + damage-prevention combo)
- `unit/075-rick-dias-red.ts` (rest + statModifier combo)

### E. `【When Linked】` on cards without a `linkCondition`

Rule 3-2-6 gates `whenLinked` triggers on the paired pilot satisfying the
unit's link condition. gd02 cards with `【When Linked】` in their printed
effect **do not set `linkCondition` in their data** (no card in gd02 defines
it), so the trigger is defined-but-ungated and any honest conversion would
fire from an invalid pairing.

Representative cards:

- `unit/008-gabthley.ts` (whenLinked deal 1 damage to rested enemy)
- `unit/009-calamity-gundam.ts` (whenLinked deal 2 damage to rested enemy)
- `unit/010-raider-gundam.ts` (whenLinked draw 1)

### G. Multi-clause conditionals: `selfIsLink`, `selfIsAttacking`,

`selfStat`, `friendlyBaseInPlay`, `oncePerTurn`, event-reactive triggered
effects

These conditions exist in the target-DSL but their interaction with specific
trigger timings is not covered by any card-level test. The effect model now
keeps those cards as `type: "triggered"` with event timings, while ongoing
state gates such as link/pair/turn status live in `activation.conditions`.
Writing a conversion would require fabricating the triggering event (e.g.
"this unit's AP is reduced by an enemy effect") which the test harness cannot
produce without another card actually doing it.

Representative cards:

- `unit/009-calamity-gundam.ts` (triggered + oncePerTurn + AP reduction event)
- `unit/010-raider-gundam.ts` (triggered + oncePerTurn + damage event)

  — the `selfIsLink` wrapper was a parser mis-read of the printed "If you
  do" inter-directive dependency. Card now uses the generic
  `dependsOnPrevious` directive flag (Wave-10 primitive) to gate the
  follow-up `draw` on the mandatory `dealDamage` actually finding a legal
  target. Sister restorations: ~~094~~ ✅, ~~098~~ ✅ (also introduces the
  `linkedUnitHasTrait` condition for the "(AEUG) Unit" gate),
  ~~125~~ ✅.
