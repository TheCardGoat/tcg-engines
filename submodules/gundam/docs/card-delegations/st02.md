### C. `duringLink` / `duringPair` combat-reactive triggers

~~Constant effects gated on `duringLink` / `duringPair` with events like

**Resolved for `duringLink` / `duringPair` event-reactive cards** — the effect
model now represents the event as `type: "triggered"` with an event timing
such as `["destroyed"]` / `["attack"]`, while the link/pair gate lives in
`activation.conditions`. Pure `type: "constant"` effects are passive only and
are no longer routed through the trigger queue.
